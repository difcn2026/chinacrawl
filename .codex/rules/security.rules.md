# 安全规则 · Security Rules

## 激活条件
当任务涉及以下内容时自动激活：
- 任何包含 `auth`、`secret`、`token`、`key`、`password`、`credential` 的路径或代码
- `.env` 或 `.env.*` 文件
- `config/` 目录下包含凭证的配置
- API 调用（任何对外 HTTP 请求）
- 用户数据（PII：姓名、手机号、邮箱、IP）

## 核心规则

### 凭证管理
1. **绝不硬编码凭证** — API Key、Token、密码一律通过环境变量或 `.env` 注入
2. **不在输出中暴露密钥** — 任何对话输出、日志、生成文件中不得包含真实凭证
3. **`.env` 不进仓库** — 确认 `.gitignore` 包含 `.env*` 和 `*.pem`
4. **密钥轮换** — 示例代码中使用的 Key 必须标明 `YOUR_KEY_HERE` 或使用占位符

### 输入安全
5. **输入校验** — 所有外部输入（用户、API 响应、文件上传）必须校验后再使用
6. **SQL 注入防范** — 数据库查询必须参数化，禁止拼接 SQL
7. **XSS 防范** — 用户生成内容输出时必须转义 HTML 特殊字符

### 依赖与权限
8. **最小权限原则** — 生成的代码/配置只申请必要的最小权限
9. **依赖审计** — 引入新依赖前检查是否有已知漏洞（`npm audit` / `pip audit`）
10. **过期依赖** — 不引入已标记 deprecated 的包

## 安全审查清单
代码审查时逐项检查：

- [ ] 无硬编码密码/Token/Key
- [ ] `.gitignore` 包含 `.env*`、`*.pem`、`credentials.*`
- [ ] API 调用使用 HTTPS
- [ ] 用户输入经过校验和清理
- [ ] 错误信息不泄露内部路径或栈信息
- [ ] 无 `eval()` / `exec()` 执行用户输入
- [ ] 文件上传限制类型和大小
- [ ] 无调试用 console.log 打印敏感信息
