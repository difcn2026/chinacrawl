# 质量规则 · Quality Rules

## 代码质量
- 函数单一职责，不超过 50 行
- 命名见名知意，避免缩写（公认缩写如 `id`、`url` 除外）
- 错误必须被处理，不允许静默吞异常
- 复杂度控制：单个函数圈复杂度 ≤ 10
- 重复容忍度：同一逻辑重复出现 3 次 → 必须抽取

## 内容质量（按阶段）

### 选题阶段
- 选题必须有差异化卖点（不能是"XX 类视频合集"）
- 目标受众画像明确（年龄、性别、兴趣）
- 7 集故事线完整可展开

### 剧本阶段
- 每集含 7 个标准节拍：钩子 → 铺垫 → 升级 → 反转 → 反应 → 收尾 → 下集钩子
- 对白符合角色人设，前后一致
- 时长估算：每分镜标注预估秒数，总和 ≤60s±5s（初稿）/ ±2s（终稿）
- 无连续 3 个分镜用同一景别（特写/近景/中景/全景需交替）

### 素材阶段
- 角色形象与 `characters.md` 一致（脸型、发型、服装、配色）
- 场景色调与项目调色板一致
- 分辨率 ≥1080p
- 素材文件名符合命名规范

### 合成阶段
- 字幕时间轴与配音对齐，误差 ≤0.3s
- 配音清晰无爆音、无背景噪音
- 总时长在目标 ±2s 内
- 片头 ≤3s，片尾 ≤5s（含关注引导）

### 发布阶段
- 标题含钩子（悬念/冲突/数字），≤25 字
- 标签 3-5 个，含 1 个品牌标签
- 描述有互动引导（关注/点赞/评论）

## 审查清单
- [ ] 产物符合阶段验收标准（见 AGENTS.md）
- [ ] 无硬编码敏感信息
- [ ] 风格与项目其他内容一致
- [ ] 输出文件路径和格式正确
- [ ] 命名符合 `naming.rules.md` 规范
- [ ] 有必要的元数据（作者/时间/版本）
