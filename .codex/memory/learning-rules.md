# 学习规则 · Learning Rules

从纠正记录和观察记录中提炼的自动化规则，逐步吸收到 `.codex/rules/` 或 `AGENTS.md` 中。

## 规则格式
- **编号**：LR-{序号}
- **来源**：纠正/观察记录的编号
- **触发条件**：什么情况下激活
- **行为**：应该怎么做
- **出现次数**：同一模式出现了几次
- **吸收状态**：待吸收 / 已吸收 / 已验证

## 吸收流程
1. 学习规则在 `learning-rules.md` 中积累
2. 同一模式 `出现次数 ≥ 3` → 提升为正式规则，移入 `.codex/rules/`
3. 验证新规则不与其他规则冲突
4. 记录到 `evolution-log.md`

## 说明
种子规则（出现次数=1）是基于项目类型预设的常见规律，不等同于已验证规则。需在实战中验证后，出现次数才会累加。
