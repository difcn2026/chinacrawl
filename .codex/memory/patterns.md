## P-001：AI 生成内容风格漂移
- **类别**：问题模式
- **描述**：连续多集剧本中，角色说话方式逐渐变化，或视觉素材风格慢慢偏离初始设定
- **根因**：每次生成时 AI 只看到当前上下文，缺乏跨集别的风格锚定
- **解决方案**：
  1. 每次生成时强制引用 `characters.md` 作为风格锚点
  2. 每 3 集做一次跨集风格一致性审查
  3. 使用固定的 prompt 模板而非每次自由发挥
- **关联规则**：LR-001, LR-002
- **出现次数**：1（种子）

## P-002：并行任务文件冲突
- **类别**：问题模式
- **描述**：多个 Agent 同时修改同一文件（如 characters.md），后完成的覆盖先完成的
- **根因**：缺乏文件锁或写前检查机制
- **解决方案**：
  1. 并行任务前明确各 Agent 的写入范围
  2. 共享文件（如 characters.md）由 Integrator 统一修改
  3. 其他 Agent 只读不写
- **关联规则**：—
- **出现次数**：1（种子）

## P-003：剧本时长估算偏差
- **类别**：问题模式
- **描述**：剧本阶段预估时长与合成后实际时长偏差超过 ±5 秒
- **根因**：分镜秒数估算依赖直觉而非经验公式
- **解决方案**：
  1. 建立经验公式：对白字数 × 0.25s + 动作描述字数 × 0.15s
  2. 修订剧本时用实际计时校准估算公式
  3. 合成后反馈实际时长到剧本文件，供后续参考
- **关联规则**：—
- **出现次数**：1（种子）
