## 三、代理软件：3proxy v0.9.6 详解

### 为什么选择 3proxy

| 对比维度 | 3proxy | Squid | CCProxy | Tinyproxy |
|---------|--------|-------|---------|-----------|
| 平台支持 | Windows/Linux | Linux 为主 | Windows | Linux 为主 |
| 内存占用 | ~5MB | ~50MB+ | ~30MB | ~10MB |
| 二进制大小 | ~450KB（单个 exe） | 多文件 | 多文件 | 单文件 |
| SOCKS5 支持 | 原生 | 需额外配置 | 支持 | 不支持 |
| 配置文件 | 纯文本，简洁 | 复杂 | GUI 为主 | 纯文本 |
| 多用户认证 | 支持 | 支持 | 支持 | 不支持 |
| 日志轮转 | 支持 D/W/M | 支持 | 支持 | 基础 |
| 适合 2GB VPS | 是 | 勉强 | 是 | 是 |

3proxy 的优势在于**极低资源消耗 + 原生 SOCKS5 + 纯文本配置 + 单文件部署**，非常适合 2GB 内存的 Windows VPS。

## 四、配置文件逐行详解

文件路径：`C:\proxy\cfg\proxy.cfg`

```
# 1. 服务模式声明（必须有，且必须放在第一行）
service

# 2. DNS 服务器配置（用于上游域名解析）
nserver 8.8.8.8
nserver 1.1.1.1

# 3. DNS 缓存大小（条数），加快重复域名解析
nscache 65536

# 4. 超时配置（共9个参数，单位秒）
#    连接超时 | 空闲超时 | 认证超时 | DNS超时 | 链接超时 | 总计超时 | 半关闭超时 | 重试间隔
timeouts 1 5 30 60 180 1800 15 60

# 5. 最大并发连接数
maxconn 1000

# 6. 日志配置
#    D = 每日轮转 (Daily)，可选 W=每周 M=每月
log C:\proxy\logs\proxy.log D

# 7. 日志格式
#    L%d-%m-%Y = 日期 | %H:%M:%S = 时间 | %z = 时区
#    %N = 连接序号 | %C = 客户端IP | %R:%r = 远程IP:端口
#    %Q = 请求字节 | %O = 发送字节 | %I = 接收字节
#    %h = 目标主机 | %T = 连接持续秒数
logformat "L%d-%m-%Y %H:%M:%S %z %N %C %R:%r %Q %O %I %h %T"

# 8. 内网监听地址（0.0.0.0 = 所有网卡）
internal 0.0.0.0

# 9. 出站IP（0.0.0.0 = 由系统路由表自动选择）
external 0.0.0.0

# 10. 用户认证配置
#     格式：users "用户名:密码类型:密码"
#     CL = 明文密码（Clear text）
#     CR = MD5 加密（crypt），格式 CR:$1$salt$hash
#     如需多用户，每行一个 users 声明
users "proxyuser:CL:StrongProxyPass2024!"

# 11. 认证模式
#     strong = 所有连接必须认证（推荐）
#     none = 不认证（仅限内网使用）
auth strong

# 12. 访问控制规则（allow/deny）
#     格式：allow/deny 源IP 目标IP 目标端口 协议 用户
#     * 表示匹配所有值
#     规则按顺序匹配，先匹配先生效
allow * * * * *

# 13. HTTP/HTTPS 代理服务（监听 3128 端口）
proxy -p3128

# 14. SOCKS5 代理服务（监听 1080 端口）
socks -p1080
```

### 配置注意事项

1. **`service` 必须放第一行**，否则作为 Windows 服务运行时会异常
2. **`external` 不宜硬编码 IP**：若 VPS 重启后 IP 变化（DHCP），硬编码会导致出站失败。`0.0.0.0` 让系统自动选择正确的网卡
3. **`nserver` 可以添加国内 DNS**：如 `114.114.114.114`（国内）或 `223.5.5.5`（阿里），加速国内域名解析
4. **`auth strong` 必须与 `users` 配合**：缺少 `users` 行会导致所有连接被拒绝
5. **`allow` 可以使用精确规则**：如 `allow 192.168.1.0/24 * * * proxyuser` 仅允许特定网段

### 5.2 注册为 Windows 服务（开机自启）

```powershell
# 第一步：停止并删除旧服务（如果存在）
sc.exe stop ProxySvc 2>$null
sc.exe delete ProxySvc 2>$null

# 第二步：创建新服务
New-Service -Name ProxySvc `
    -BinaryPathName '"C:\proxy\bin64\3proxy.exe" "C:\proxy\cfg\proxy.cfg"' `
    -StartupType Automatic `
    -DisplayName "3proxy Proxy Server"

# 第三步：配置失败自动重启
#   reset=86400   → 24小时内重置失败计数
#   restart/5000  → 第一次失败后5秒重启
#   restart/10000 → 第二次失败后10秒重启
#   restart/30000 → 第三次失败后30秒重启
sc.exe failure ProxySvc reset=86400 actions=restart/5000/restart/10000/restart/30000

# 第四步：启动服务
Start-Service ProxySvc

# 第五步：验证服务状态
Get-Service ProxySvc
```

## 七、Windows Defender 排除配置

### 问题说明

3proxy.exe 可能被 Windows Defender 误报为病毒/潜在有害软件（PUA），导致：
- exe 文件被隔离/删除
- 无法复制或运行
- 服务无法启动

### 解决方法

```powershell
# 方案一：添加排除路径（推荐）
Add-MpPreference -ExclusionPath C:\proxy

# 验证排除列表
Get-MpPreference | Select-Object -ExpandProperty ExclusionPath

# 方案二：添加排除进程
Add-MpPreference -ExclusionProcess "C:\proxy\bin64\3proxy.exe"

# 方案三：临时禁用实时保护（重启后自动恢复）
Set-MpPreference -DisableRealtimeMonitoring $true

# 恢复实时保护
Set-MpPreference -DisableRealtimeMonitoring $false

# 查看保护状态
Get-MpComputerStatus | Select-Object *Enabled, *Running
```

### 补充说明

- 排除路径后，该目录下所有文件不再被扫描
- 建议仅排除 `C:\proxy` 而非整个 C 盘
- 如果 exe 已被删除，需要先关闭实时保护，重新解压，再添加排除

## 九、故障排查手册

| 症状 | 可能原因 | 排查步骤 |
|------|---------|---------|
| 端口不通（Connection refused） | 代理服务未启动 | `Get-Service ProxySvc` 查看服务状态；`netstat -ano \| findstr 3128` 查看端口监听 |
| 端口不通但服务在运行 | 防火墙拦截 | `netsh advfirewall firewall show rule name="3proxy HTTP"` 检查规则是否存在 |
| 服务无法启动 | Defender 误杀 exe | 检查 `C:\proxy\bin64\3proxy.exe` 是否存在；添加 Defender 排除路径 |
| 服务启动后又停止 | 配置文件错误 | 前台运行 `cmd /c "C:\proxy\bin64\3proxy.exe C:\proxy\cfg\proxy.cfg"` 查看错误输出 |
| 认证失败（407 / Auth failed） | 密码错误或用户不存在 | 检查 `proxy.cfg` 中 `users` 行格式；确认 `auth strong` 已配置 |
| DNS 解析失败 | DNS 服务器不可达 | 测试 `nslookup google.com 8.8.8.8`；在 `proxy.cfg` 中更换 `nserver` |
| 502 Bad Gateway | VPS 自身网络问题 | 在 VPS 上直接 `curl https://github.com` 测试网络连通性 |
| 连接速度慢 | 跨国网络延迟 | 正常现象，ping 150-200ms；可尝试更换 `nserver` 为更快 DNS |
| 大量 TIME_WAIT | 短连接过多 | 正常现象，Windows 默认 120s 回收；`maxconn 1000` 已配置充足连接池 |
| 重启后代理失效 | 未注册为 Windows 服务 | 执行"五、启动与日常管理"中的服务注册步骤 |

## 十一、日志管理

### 11.1 日志格式

每行日志包含：

```
日期 时间 时区 连接序号 客户端IP 远程IP:端口 请求字节 发送字节 接收字节 目标主机 持续时间(秒)
```

示例：

```
04-06-2026 14:32:01 +0800 00001 192.168.1.100 github.com:443 512 10240 204800 github.com 12
```

### 11.2 日志轮转

配置 `log C:\proxy\logs\proxy.log D` 中：
- `D` = 每日轮转（生成 `proxy.log.YYYY.MM.DD`）
- `W` = 每周轮转
- `M` = 每月轮转

旧日志文件会保留在 `logs` 目录，需手动清理。

### 11.3 日志分析

```powershell
# 统计今日连接数
Get-Content C:\proxy\logs\proxy.log | Measure-Object -Line

# 统计访问最多的目标
Get-Content C:\proxy\logs\proxy.log | ForEach-Object { ($_ -split '\s+')[8] } | Group-Object | Sort-Object Count -Descending | Select-Object -First 10

# 查找失败连接（没有持续时间或流量为0的行）
Select-String -Path C:\proxy\logs\proxy.log -Pattern " 0 0 0 " | Select-Object -First 20
```

## 十三、3proxy 高级功能

### 13.1 流量限制

在配置文件中添加：

```
# 每用户带宽限制（字节/秒）
bandwidth 1048576  # 1MB/s

# 或使用 TrafficPlugin 按用户限速
plugin C:\proxy\bin64\TrafficPlugin.dll traffic.conf
```

### 13.2 SSL/TLS 加密代理

```
# 启用 SSL 加密（客户端到代理之间的连接加密）
plugin C:\proxy\bin64\SSLPlugin.dll ssl.conf
```

需要先配置 SSL 证书。

### 13.3 透明代理（TPROXY）

```
# 透明代理模式（需配合 iptables/路由规则，主要用于 Linux）
# Windows 上较少使用
plugin C:\proxy\bin64\TransparentPlugin.dll tproxy.conf
```

### 13.4 父级代理链（proxy chaining）

如果需要通过另一个上游代理访问：

```
# parent 命令格式：parent 端口 上游IP 上游端口 用户名 密码
parent 1000 proxy2.example.com 3128 user2 pass2
```

## 十四、常见问题 FAQ

**Q: 密码中的感叹号 `!` 在命令行中怎么处理？**
A: 在 PowerShell 中用单引号包裹，如 `'StrongProxyPass2024!'`。在 curl 的 URL 中，感叹号是合法字符，无需转义。在批处理/cmd 中需要用 `^` 转义。

**Q: 代理能看 YouTube 吗？**
A: 能。已验证通过 SOCKS5 代理可正常播放 YouTube 视频。VPS 在美国，网络不受限制。

**Q: 代理速度和 VPS 带宽有关吗？**
A: 有关。2GB VPS 通常含 1Gbps 共享带宽或固定带宽（视套餐而定），建议查看 RackNerd 控制面板中的带宽使用情况。

**Q: 多个客户端同时使用会怎样？**
A: 3proxy 配置了 `maxconn 1000`，可支持 1000 个并发连接。实际约可支撑 50-100 个用户同时轻度使用。

**Q: 如果 VPS 重启了代理会怎样？**
A: 如果已注册为 Windows 服务（`StartupType Automatic`），VPS 重启后代理会自动启动。如果未注册服务，需要手动启动。

**Q: 能添加更多端口吗？**
A: 可以。在配置文件底部添加更多 `proxy -p端口号` 或 `socks -p端口号` 行，然后在防火墙中放行对应端口。

**Q: 如何监控代理使用情况？**
A: 方法1：查看日志文件；方法2：使用 `netstat -ano | findstr "3128 1080"` 查看当前连接数；方法3：启用 TrafficPlugin 统计流量。
