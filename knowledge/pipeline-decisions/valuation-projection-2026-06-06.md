> 注意：这个数字包含传统代理+抓取服务。AI 驱动的搜索+抓取+LLM 提取是其中增速最快的子赛道（30-40% YoY）。

### 中国 Web Scraping 市场推算

全球市场没有按地区拆分的权威数据。基于以下逻辑推算：

```
全球 web scraping 市场: ~$2B (2026)
中国占全球互联网用户 ~20%，但 web scraping 商业化程度较低
→ 中国市场份额: 保守 5-8%，乐观 12-15%
→ 中国市场: $100-300M (¥7-22亿)

加上中国独特的搜索引擎壁垒:
→ 百度/360/搜狗搜索抓取 = 海外竞品无法参与的增量市场
→ 实际可触达市场可能更大，因为搜索+LLM不只是"抓取"，更是"AI数据基础设施"
```

## 四、关键变量与杠杆

### 上杠杆的因素

| 杠杆 | 效果 | 可行性 |
|------|------|:--:|
| 被中国 AI 框架（Dify/FastGPT）官方推荐 | 付费转化率翻倍 | 中（需要深度集成+关系） |
| 中文搜索独占的 PR 传播 | 开源用户 5k→20k | 高（故事好讲） |
| 推出「短剧选题搜索」垂直场景 | 差异化壁垒加深 | 极高（我们已有短剧生态） |
| 被收购（Firecrawl 进军中国时收购我们） | 一次性退出 $5-15M | 低-中 |

### 压杠杆的因素

| 风险 | 影响 | 概率 |
|------|------|:--:|
| Firecrawl 12-18月内加百度后端 | 窗口关闭，护城河消失 | 中 |
| 百度/360 大规模封锁 SearXNG | 产品核心功能失效 | 中 |
| 中国 web scraping 法律收紧 | 合规成本飙升 | 低-中 |
| 开源社区无人问津 | 冷启动失败 | 中 |

## 五、结论

### 一句话版

> **Firecrawl 值 ~¥7-11亿（$100-150M）。xhls_scraper 如果做到基准档，3年后值 ~¥4000万（$5.6M）；如果乐观档，值 ~¥1.5亿（$21M）。差距 3-13 倍，但我们的边际成本是零，启动资金也是零。**

### 现实版

这不是一个「变成独角兽」的故事。这是一个：

```
零资金 + 已有代码 + 独占切角 → 年收入 ¥40-130万 → 估值 ¥400-1500万
```

如果被收购（Firecrawl 进中国时），可能一次性退出 $5-15M。

如果独立做到中国 web scraping SaaS 第一梯队，天花板大约 $20-50M 估值。

**对于一个人的副业项目，投资回报比极高；对于 VC 路线的创业公司，故事太小。**
