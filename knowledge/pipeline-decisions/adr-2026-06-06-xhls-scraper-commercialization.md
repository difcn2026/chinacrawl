## 背景

用户提出：`xhls_scraper.py` 是否可以直接放入市场进行商业变现？

`xhls_scraper.py`（~1040行单文件）是 XHLS v3.0 自研的 Firecrawl 平替，具备 11 项能力：
scrape / map / search / monitor / monitor-ai / session / crawl / download / interact / extract / extract-llm。

核心依赖链：`SearXNG 私有实例 (47.236.24.76:9999)` → `httpx + trafilatura` → `Playwright (浏览器交互)` → `Ollama + qwen2.5:7b (LLM提取)`

**关键差异化**：自有 SearXNG 实例实现零边际成本无限调用，不依赖任何外部付费 API。

---

## 决策问题

`xhls_scraper.py` 当前形态是否具备直接商业变现条件？如果否，是否存在可行路径？

---

## 方案对比

### 方案A：直接挂 marketplace 卖源码

- **优势**：零额外投入，即插即卖
- **劣势**：单文件脚本无产品壁垒，竞品泛滥，客单价必然极低（<$10）
- **风险**：无持续收入，一次交易后无复购

### 方案B：封装为 SaaS API 服务

- **优势**：对标 Firecrawl $19-299/月定价，MRR 模型清晰
- **劣势**：需要多租户隔离、计费系统、API 网关、SLA 保障、运维团队——与当前单文件脚本差距 2-3 人月工作
- **风险**：Firecrawl 先发优势 + 品牌认知，正面竞争无胜算

### 方案C：开源核心 + 中国本土化托管（推荐）✅

- **优势**：
  - Firecrawl 等竞品在中国不可用（GFW + 不支持百度/搜狗/微信搜索后端）
  - SearXNG 可配置中文搜索引擎，填补真空
  - qwen2.5:7b 中文 LLM 提取是端到端中文处理独有能力
  - 开源建社区信任 → 托管服务变现，路径清晰
- **劣势**：需要 1-3 个月产品化投入
- **风险**：中国市场付费习惯培养周期长

| 方案 | 投入 | 天花板 | 护城河 | 推荐 |
|------|:--:|:--:|:--:|:--:|
| A: 源码直销 | 极低 | 极低 | 无 | ❌ |
| B: SaaS 正面对标 | 极高 | 中高 | 弱 | ❌ |
| C: 开源+中国托管 | 中 | 中高 | 强（中文搜索独占） | ✅ |

---

## 决策

**选择方案C：开源核心 + 中国本土化托管**

### 理由链

```
GFW 阻隔海外 API → 中国开发者需要境内可用的搜索抓取链
Firecrawl 不支持百度/搜狗/微信搜索 → 中文内容搜索是真空地带
SearXNG 可配百度/360 后端 → 私有实例有中文搜索能力
qwen2.5:7b 中文提取 → 端到端中文内容处理，竞品无此能力
```

### 四阶段路线图

| 阶段 | 动作 | 预估时间 |
|------|------|:--:|
| 1 | 开源 `xhls_scraper`，README 突出「零成本 + 中文搜索 + 本地 LLM」三合一卖点 | 1-2周 |
| 2 | 封装 Docker 一键部署（SearXNG + Ollama + scraper），降低试用门槛 | 1周 |
| 3 | 境内云主机托管版（阿里云/腾讯云），按月收费，卖「免部署 + 中文搜索」 | 1月 |
| 4 | 社区版免费 + Pro 版（更多并发/更深抓取/企业 SSO） | 持续 |

### 定价锚点

Firecrawl 在中国不可用 → 掌握定价权。

- 个人版：¥29/月
- 团队版：¥199/月
- 企业版：¥999/月（含私有部署 + SLA）

## 附：SearXNG 多租户架构方案

### 问题

SearXNG 单实例无法直接支持多租户 SaaS，因为：

1. **IP 风控连坐**：百度/360 反爬按出口 IP 限流。一个用户滥用 → 同 IP 所有用户被封
2. **搜索偏好隔离**：不同租户需要不同搜索引擎组合（企业要百度+360+搜狗，个人只要百度）
3. **计量计费**：单实例日志混在一起，无法按租户统计用量

### 方案

**推荐：共享实例 + 出口 IP 池轮转**

```
                    ┌─────────────────┐
                    │  API Gateway     │
                    │  per-tenant      │
                    │  rate limit      │
                    └──────┬──────────┘
                           │
                    ┌──────▼──────────┐
                    │  xhls_scraper   │
                    │  (单实例)        │
                    └──────┬──────────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │ SearXNG  │ │ SearXNG  │ │ SearXNG  │
        │ 实例 1   │ │ 实例 2   │ │ 实例 3   │
        │ 引擎:百度│ │ 引擎:360 │ │ 引擎:必应│
        │ IP: X.1  │ │ IP: X.2  │ │ IP: X.3  │
        └──────────┘ └──────────┘ └──────────┘
              │            │            │
              └────────────┼────────────┘
                           │
                    出口 IP 池轮转
                  (5-10个弹性公网IP)
```

**成本（阶段2）**：SearXNG ×3（~¥300/月）+ 弹性公网IP ×5（~¥100/月）= **~¥400/月**

**实施复杂度**：Docker Compose 多实例 + Nginx upstream 分配，一天可完成。
