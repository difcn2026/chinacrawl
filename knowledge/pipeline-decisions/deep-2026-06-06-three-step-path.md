## 一、「Firecrawl 在中国做不了的事」——五层拆解

这个结论不是一句口号。把它拆成五层，每一层都是一个独立的市场真空。

### 第1层：搜索覆盖面（最致命）

中国搜索引擎市场份额（2025年6月，StatCounter 全设备）：

| 搜索引擎 | 市场份额 | Firecrawl 是否支持 |
|----------|:--:|:--:|
| 百度 | 50.7% | ❌ |
| 必应（中国版 cn.bing.com） | 24.2% | ❌ (仅国际版) |
| 好搜（360搜索） | 17.0% | ❌ |
| Yandex | 4.3% | ❌ |
| 搜狗 | 1.7% | ❌ |
| Google | 1.4% | ✅ |

**结论：Firecrawl 只能覆盖中国 1.4% 的搜索流量。xhls_scraper 通过 SearXNG 可配置百度/360/必应/搜狗后端，覆盖 ~93.6%。**

SearXNG 支持的搜索引擎中，中国可用的：
- `baidu` — 百度网页搜索
- `bing` — 可指定 `bing.com` 或 `cn.bing.com`
- `360search` — 好搜
- `sogou` — 搜狗（SearXNG 部分支持）
- `zhihu` — 知乎站内
- `bilibili` — B站站内

### 第2层：基础设施可达性

```
Firecrawl 用户在中国大陆的困境：
  firecrawl.dev → GFW 未直接墙，但 API endpoint 不稳定
  api.firecrawl.dev → 间歇性超时，延迟 2-5s+
  支付 → Stripe 不支持支付宝/微信，企业采购极难
  合规 → 数据出境风险，中国企业不敢用海外 SaaS 抓国内数据

xhls_scraper 的优势：
  SearXNG → 阿里云/腾讯云境内部署，延迟 <50ms
  支付 → 支付宝/微信原生支持（若做托管版）
  合规 → 数据不出境，SearXNG 实例在境内
```

### 第3层：中文内容理解

| 能力 | Firecrawl | xhls_scraper |
|------|:--:|:--:|
| 原始抓取 | ✅ (trafilatura/jina) | ✅ (同) |
| 中文搜索 | ❌ (无中文引擎) | ✅ (SearXNG + 百度/360) |
| **LLM 中文提取** | 需接 OpenAI (英文优先) | ✅ qwen2.5:7b (中文原生) |
| 中文分词/关键词 | 依赖外部 | Ollama 本地完成 |
| 微信/小红书/抖音内容 | ❌ | Playwright 浏览器可渲染 |

### 第4层：成本结构

| | Firecrawl | xhls_scraper |
|------|:--:|:--:|
| 搜索 | $19/月起 (1000 credits) | **¥0** (自有 SearXNG) |
| 抓取 | 按 credit 计费 | **¥0** (trafilatura/jina 免费) |
| LLM 提取 | 走 OpenAI API (按 token) | **¥0** (Ollama 本地 qwen) |
| 边际成本/次 | ~$0.02-0.05 | **~¥0.001 (仅电费+带宽)** |

### 第5层：生态整合

Firecrawl 是西方开发者生态的一环（对接 LangChain、OpenAI、GitHub Actions）。xhls_scraper 的生态机会在：

- **中国 AI 开发者**：对接 Dify、FastGPT、LangChain 中文社区
- **短剧/内容创作者**：选题调研、竞品监控、热点追踪
- **跨境电商**：亚马逊/速卖通评论抓取、竞品 Listing 监控
- **企业合规爬虫**：工商信息、司法数据、招投标信息

## 二、三段跳路径逐段拆解

### 🔵 阶段1：开源包装（1-2周）

**目标**：不是「把代码扔到 GitHub」，而是建立开发者心智——「西有 Firecrawl，东有 xhls_scraper」。

**具体动作**：

```
1. 仓库结构
   xhls-scraper/
   ├── README.md          ← 中英双语，突出"11合1+零成本+中文搜索"
   ├── README_zh.md       ← 中文详细文档
   ├── xhls_scraper.py    ← 核心引擎（单文件，pip install 即用）
   ├── docker-compose.yml ← SearXNG + Ollama 一键部署
   ├── examples/          ← 场景化示例
   │   ├── market_research.py    # 竞品调研
   │   ├── trend_monitor.py      # 热点监控
   │   └── content_extract.py    # 结构化提取
   ├── tests/
   └── docs/
       ├── vs-firecrawl.md       # 对比表
       └── china-search.md       # 中文搜索引擎配置指南
```

2. **分发策略**
   - GitHub 主仓库 + Gitee 镜像（中国开发者首选）
   - PyPI 发布 `pip install xhls-scraper`
   - Docker Hub 发布一键部署镜像
   - 在 Dify/FastGPT 社区发「零成本搜索插件」教程

3. **传播钩子**
   - 标题：「Firecrawl 在中国只能覆盖 1.4% 的搜索——这个开源替代覆盖 93%」
   - 对比图：Firecrawl vs xhls_scraper 搜索覆盖率饼图
   - Demo 视频：30 秒从零部署到第一条搜索结果

**关键指标**：
- GitHub Stars >500（首月）
- Gitee 推荐项目
- 被至少 1 个中国 AI 框架（Dify/FastGPT）收录为推荐插件

**风险**：
- Firecrawl 可能随后推出中国版 → 窗口期有限
- 代码质量被社区 scrutiny → 需要先做一轮重构（从 ~1040 行单文件拆为模块）

### 🟢 阶段2：境内云托管（1个月）

**目标**：开源用户中的一部分不想自建 SearXNG + Ollama → 付费用托管 API。

**架构**：

```
用户 (浏览器/API) 
    │
    ▼
API Gateway (Nginx/Kong)
    │
    ▼
┌──────────────────────────────┐
│  多租户 xhls_scraper 服务    │
│  ┌────────┐ ┌────────┐      │
│  │租户A   │ │租户B   │ ...  │
│  │rate:10 │ │rate:50 │      │
│  └────────┘ └────────┘      │
│        │          │          │
│        ▼          ▼          │
│  ┌──────────────────────┐   │
│  │  SearXNG 集群 (多实例)│   │
│  │  • 百度后端 ×2       │   │
│  │  • 360后端 ×2        │   │
│  │  • Bing后端 ×1       │   │
│  └──────────────────────┘   │
│  ┌──────────────────────┐   │
│  │  Ollama 集群 (GPU)    │   │
│  │  • qwen2.5:7b ×2     │   │
│  └──────────────────────┘   │
└──────────────────────────────┘
```

**定价模型**：

| 版 | 月费 | 搜索次数 | 抓取次数 | LLM提取 | 并发 |
|------|:--:|:--:|:--:|:--:|:--:|
| 免费 | ¥0 | 100/天 | 50/天 | 10/天 | 1 |
| 个人 | ¥29 | 1000/天 | 500/天 | 100/天 | 3 |
| 团队 | ¥199 | 10000/天 | 5000/天 | 1000/天 | 10 |
| 企业 | ¥999 | 无限* | 无限* | 无限* | 50 |

**成本估算（阿里云 ECS）**：

| 资源 | 规格 | 月费 |
|------|------|:--:|
| SearXNG 实例 ×3 | 2C4G | ~¥300 |
| Ollama GPU 实例 ×2 | T4 GPU 16GB | ~¥2000 |
| API Gateway | 2C4G | ~¥200 |
| 带宽 | 按量 | ~¥500 |
| **合计** | | **~¥3000/月** |

**盈亏平衡点**（假设 ¥3000/月固定成本）：
- 需 ~104 个个人版用户，或 15 个团队版用户
- 实际上 SaaS 通常转化率 2-5%，需 ~2000-5000 开源用户基础

**关键动作**：
- 境内服务器部署（阿里云杭州/北京节点）
- ICP 备案（如涉及网页服务）
- 支付宝/微信支付接入
- API Key 管理 + 用量监控后台

### 🟣 阶段3：企业 Pro 版（持续演进）

**目标**：从「工具」升级为「平台」，卖给有合规需求的企业客户。

**Pro 版差异化功能**：

| 功能 | 社区版 | Pro 版 |
|------|:--:|:--:|
| 私有化部署 | ❌ | ✅ 一键部署到客户 VPC |
| SLA 保障 | ❌ | ✅ 99.9% |
| SSO/SAML | ❌ | ✅ |
| 审计日志 | ❌ | ✅ 全量操作记录 |
| 自定义搜索引擎 | ❌ | ✅ 企业内部搜索源 |
| 数据驻留 | 公共集群 | 专属实例 |
| 合规认证 | 无 | 等保二级/ISO27001 |
| 专属客服 | 社区 | 7×24 企业微信 |

**目标客户画像**：

1. **AI 创业公司**（Dify/FastGPT 用户）：需要搜索增强 RAG，不想付海外 API 费
2. **跨境电商 SaaS**：抓取竞品 Listing/评论，中文内容分析
3. **舆情/市场研究公司**：百度/360/搜狗搜索结果批量采集
4. **企业内部知识管理**：聚合搜索 + 文档抓取 + LLM 摘要

**定价**：¥50,000-200,000/年（企业私有部署 + SLA）

## 三、风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|:--:|:--:|------|
| Firecrawl 推出中国版 | 中 | 高 | 抢占先发优势，建立社区壁垒 |

| 短剧项目占用精力 | 高 | 中 | 严格时间盒管理，阶段1可在周末完成 |

## 四、时机判断

**为什么是现在？**

1. **Firecrawl 刚被收购/调整**（Reddit 社区抱怨「Firecrawl stopped being useful」），用户心态松动
2. **中国 AI 应用爆发**（Dify 3万+ GitHub Stars，FastGPT 1.5万+），搜索增强 RAG 是刚需
3. **SearXNG 成熟度**：2.0+ 版本稳定，Docker 部署一行命令
4. **qwen2.5:7b 中文能力**：中文提取质量已达可用线
5. **我们已有完整可用的 11 合 1 引擎**——不是 PPT，是能跑的东西

**窗口期估计**：12-18 个月。Firecrawl 或其他竞品可能在此期间推出中国版。
