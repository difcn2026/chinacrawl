## 第1层：技术护城河 — SearXNG 中文搜索引擎配置库

### Firecrawl 要加百度搜索的代价

```
逆向百度搜索页面结构       → 2-3 人月
解决百度反爬 (验证码/IP限流/行为检测) → 持续猫鼠游戏
适配百度搜索结果格式       → 与 Google/Bing 完全不同
维护更新 (百度每月都可能改版) → 持续人力投入

× 5 个中文引擎 (百度/360/搜狗/必应中国/微信搜一搜)
= 至少 6-12 人月工程投入 + 持续维护团队
```

### 我们的壁垒动作

| 动作 | 壁垒效果 |
|------|------|
| 中文引擎配置文档化、模板化 | 降低模仿者门槛，但提高切换成本（用户锁定在我们的配置） |
| 建立反爬策略库（UA轮转、IP池、请求间隔、验证码识别） | 不让竞品直接抄 SearXNG 默认配置就能用 |
| 持续追踪各引擎反爬升级 | 积累隐性知识，越追越难被追上 |

**本质**：不是代码壁垒，是**持续投入的维护成本壁垒**。谁先进来谁积累经验，后来者追不上。

## 第2层：数据飞轮

```
用户搜索 "短剧爆款选题" → 搜索结果
    → 用户用 LLM 提取结构化数据
        → 提取模板/规则沉淀到公共库
            → 新用户搜索类似关键词时直接复用模板
                → 提取质量更高 → 用户更多 → 循环加速
```

### 可沉淀的数据资产

| 数据资产 | 来源 | 壁垒强度 |
|------|------|:--:|
| 中文搜索引擎配置（已验证可用） | 初期研发 | 高 |
| 反爬规则库（UA/延迟/IP策略） | 持续运营 | 极高 |
| LLM 提取 prompt 模板库 | 用户贡献 | 中 |
| 中文网站分类爬取策略 | 社区贡献 | 中 |

## 第3层：社区网络效应

```
          ┌──────────────┐
          │ Dify 集成     │ ← 中国最大 AI 应用框架
          │ FastGPT 集成  │ ← 第二大
          │ LangChain 中文│ ← 国际桥头堡
          └──────┬───────┘
                 │
    用户在这些框架里选「搜索引擎插件」
        → 看到 xhls_scraper（中文搜索）vs Firecrawl（海外搜索）
            → 99% 中国用户选 xhls_scraper
                → 更多用户 → 更多集成 → 壁垒加深
```

**本质**：不是技术壁垒，是**集成深度壁垒**。一旦进入 Dify 官方推荐插件列表，竞品替换的不只是产品更好，而是要让 Dify 换推荐——比技术竞争难得多。

## 第4层：合规壁垒

| 壁垒 | 说明 |
|------|------|
| ICP 备案 | 托管版必须备案，海外竞品做不到 |
| 数据出境合规 | 中国客户用海外 SaaS 抓国内数据 = 违法风险 |
| 等保认证 | 企业客户要求等保二级以上，自托管天然合规 |
| 内容安全审查 | 搜索结果可能含敏感内容，需要过滤机制 |

**本质**：法律法规天然排斥海外竞品进入中国市场。我们不构建这个壁垒，它已经在那——要做的是**在合规框架内合法运营**，让壁垒成为对手的门槛而非我们的镣铐。

## 第6层：许可证护城河 — AGPL v3

```
AGPL v3 的效果：
  ✓ 任何人可以看代码、改代码
  × 但如果云托管给别人用（SaaS 模式），必须开源所有修改
  × 竞争对手不能直接拿代码做托管版竞争
  × 大厂不能 fork → 改改 → 云服务上线（阿里云/腾讯云）
  ✓ 我们自己可以做托管版（因为是版权持有者）
```

这是开源商业模式的标准玩法——MongoDB、Elasticsearch 都这么干。
